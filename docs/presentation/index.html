Conteneurisation de  l'API

Pourquoi conteneuriser ?

  • Encapsule l'application, runtime complet.
  • Permet l'execution identique peu importe la machine.
  • Déploiement facile.

JIB

Pourquoi ?

  • Intégré a Maven
  • Sans dockerfile
  • Optimisé
  • Facile a intégrer

Text

Configuration

application.properties :

  • quarkus.container-image.build=true → Active la génération d'image
  • quarkus.jib.docker-executable-name=podman → Utilisation de Podman
  • quarkus.container-image.name=forfaits-opt-nc → Nom de l’image

OpenAPI

Annotations dans le code source

Specifications YAML, JSON

  • @Path("/offres") → Définit l’URL de l’endpoint

  • @GET → Utilise la méthode HTTP GET

  • @Produces(MediaType.APPLICATION_JSON) → Réponse au format JSON

  • @Operation → Décrit l’action de l’endpoint

  • @Parameter → Définit les paramètres d’entrée (ex. : ID du forfait)

  • @APIResponse → Spécifie les réponses possibles (codes HTTP, exemples)

  • @Tag → Catégorise les endpoints pour la documentation

name: Deploy
on:
  release:
    types: [published]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Install NodeJS
        uses: actions/setup-node@v3
        with:
          node-version: '18.19.1'

      - name: Install Redocly
        run: npm i -g @redocly/cli@latest

      - name: Build API HTML page
        id: build_api_html_page
        run: |
          redocly build-docs openapi/openapi.yaml --output=./docs/api/index.html

      - name: ⚙️ Setup Github Pages
        uses: actions/configure-pages@v5
      - name: 📤 Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './docs/api'
      - name: 🚀 Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@main

Gestion erreur 404 :

Importance de la gestion d'erreur 404 :

  • Optimisation de l'expérience utilisateur
  • Sécurité : Empêcher l'exposition d'informations sensibles.

Comment ?

  • Bloc try-catch :

 

    @PathParam("id") String id) {
        ForfaitM forfaitM;
        try {
            forfaitM = entityManager.createQuery("SELECT fm FROM ForfaitM fm WHERE fm.id = :id", ForfaitM.class)
                                    .setParameter("id", id)
                                    .getSingleResult();
            return Response.ok(forfaitM).build();
        } catch (Exception e) {
            return Response.status(Response.Status.NOT_FOUND)
                           .entity("Forfait avec ID '" + id + "' non trouvé")
                           .type(MediaType.TEXT_PLAIN)
                           .build();
        }
    }

    @Test
    public void testGetAllForfaitsGammeM() {
        given()
          .when().get("/offres/forfait-m")
          .then()
             .statusCode(200)
             .body("$", notNullValue())
             .body("$", hasSize(4))

    @Test
    public void testGetForfaitMById() {
        given()
          .pathParam("id", "forfait-m-1")
          .when().get("/offres/forfait-m/{id}")
          .then()
             .statusCode(200)
             .body("id", equalTo("forfait-m-1"))
             .body("prix", equalTo(1000.0f)); 
    }
    @Test
    public void testGetForfaitMById_NotFound() {
